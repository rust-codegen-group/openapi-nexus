# Target Output Spec вЂ” Phase 0

This is the fixed target for the sigil-stitch migration. It shows what generated
code should look like *after* the rewrite, side-by-side with what it looks like
today. Golden tests will be updated to match these targets during Phase 3вЂ“4.

**This is a breaking change.** Output bytes will differ. Users who depend on
`FromJSON`/`ToJSON`/`instanceOfX` helpers, `ApiXxxRequest` wrapper interfaces,
or the barrel `index.ts` must migrate. Migration notes live alongside the v0.2.0
release notes (Phase 5).

## Locked decisions

| # | Decision | Rationale |
|---|---|---|
| 1 | Field terminator is `;` (not `,`) | Standard idiomatic TypeScript; interfaces and types |
| 2 | Drop `FromJSON` / `ToJSON` / `FromJSONTyped` / `ToJSONTyped` | Interfaces are structural; runtime conversion is noise when field names already match |
| 3 | Drop `instanceOfX` | Structural checks in TS are a code smell; use discriminators or validators |
| 4 | Drop `XxxPropertyValidationAttributesMap` | Dead data; validation belongs in a schema library (see #6) |
| 5 | Drop `/* tslint:disable */` and `/* eslint-disable */` headers | tslint is deprecated; modern projects configure per-file if needed. Keep `@generated` header. |
| 6 | Validator is opt-in via `--validator zod\|valibot\|none` (default `none`) | Schemas ship as siblings (`Pet.schema.ts`), never force-imported. User calls `parsePet(json)` at trust boundaries. |
| 7 | No `models/index.ts` barrel | Barrels defeat tree-shaking and slow down IDEs at scale. Callers import from specific files. |
| 8 | Every field is `readonly` | Generated DTOs are immutable views of wire data. Mutation is a code smell; wrap in your own type if you need it. |
| 9 | Runtime is embedded, ~80 lines target (was 450) | Current runtime carries dead abstractions (`BaseAPI`, `JSONApiResponse`, `BlobApiResponse`, вЂ¦). Replace with a single `request()` helper. |
| 10 | Per-tag API classes + top-level `Client` wrapper | Ergonomic `new Client({ baseUrl }).pets.getPetById(...)` replaces the per-file import dance. |
| 11 | `Client` name derived from `info.title` via `heck::ToPascalCase` + `Client` suffix; CLI `--client-name` overrides | `"Studio AMS вЂ” ASM"` в†’ `StudioAmsAsmClient`. Non-ASCII / unicode dashes stripped. |
| 12 | Errors throw `ApiError` (extends `Error`), with `ApiNetworkError` and `ApiResponseError` subclasses; typed `.body` on response errors | Idiomatic JS вЂ” callers use `try/catch` and `instanceof ApiError`. No more union-of-ApiResponse return types. |
| 13 | Methods take an options object; path params, query, body are named keys | `client.pets.getPetById({ petId: 7 })` вЂ” consistent regardless of param count |

Everything else (filenames, PascalCase models, snake_case wire в†” camelCase TS
mapping when schemas disagree) stays.

## 1. Model file вЂ” `Pet.ts`

### Before (90 lines)

```ts
/* tslint:disable */
/* eslint-disable */
/**
 * Petstore API
 * ...
 * NOTE: This file is auto generated by openapi-nexus.
 */
import { type Category, CategoryFromJSON, CategoryFromJSONTyped, CategoryToJSON, instanceOfCategory } from './Category';
import { type PetStatus, PetStatusFromJSON, PetStatusFromJSONTyped, PetStatusToJSON } from './PetStatus';
import { type Tag, TagFromJSON, TagFromJSONTyped, TagToJSON, instanceOfTag } from './Tag';

/** Pet model */
export interface Pet {
  category?: null | Category,
  id?: number | null,
  name: string,
  photoUrls: Array<string>,
  status?: null | PetStatus,
  tags?: null | Array<Tag>,
}

export function instanceOfPet(value: unknown): value is Pet { ... }
export function PetFromJSON(json: any): Pet { ... }
export function PetFromJSONTyped(json: any, ignoreDiscriminator: boolean): Pet { ... }
export function PetToJSON(value?: Pet | null): any { ... }
export function PetToJSONTyped(value?: Pet | null, ignoreDiscriminator: boolean = false): any { ... }

export const PetPropertyValidationAttributesMap: { ... } = { };
```

### After (~20 lines)

```ts
/**
 * @generated by openapi-nexus. Do not edit.
 *
 * Petstore API вЂ” 1.0.0
 * This is a sample Pet Store Server based on the OpenAPI 3.1 specification
 */
import type { Category } from './Category';
import type { PetStatus } from './PetStatus';
import type { Tag } from './Tag';

/** Pet model */
export interface Pet {
  readonly category?: Category | null;
  readonly id?: number | null;
  readonly name: string;
  readonly photoUrls: readonly string[];
  readonly status?: PetStatus | null;
  readonly tags?: readonly Tag[] | null;
}
```

**Notes:**
- `readonly` on every field. Arrays become `readonly T[]`.
- `Array<T>` в†’ `T[]` for primitives / simple refs (less visual weight). Keep `Array<Map<...>>`-style only for complex inner types.
- Optional-with-nullable stays as `T | null` with `?:` вЂ” TS handles this natively.
- `type` imports only (no value imports) because no runtime helpers ship.
- Wire-name vs TS-name mismatches (e.g., `photo_urls` в†” `photoUrls`): still applied during generation. No runtime translation layer вЂ” callers get camelCase in TS and that's what goes on the wire **only if** the spec's wire name already matches. **If wire name differs**, emit it as-is (stay faithful to the spec) вЂ” we'll document this in the migration guide.

### Enum вЂ” `PetStatus.ts`

**Before (40 lines)** вЂ” enum + `FromJSON`/`FromJSONTyped`/`ToJSON`/`ToJSONTyped`.

**After (~5 lines):**

```ts
/**
 * @generated by openapi-nexus. Do not edit.
 */

/** Pet status in the store */
export type PetStatus = 'available' | 'pending' | 'sold';
```

String-literal union type alias. No `enum` (TS-only runtime construct, not
erasable-syntax compatible), no `as const` companion object (callers who want
`PetStatus.Available` can wrap it themselves; generated clients ship types,
not conveniences). Minimal surface, cleaner diffs, better tree-shaking.

## 2. Validator schemas (opt-in) вЂ” `Pet.schema.ts`

Emitted **only** when `--validator zod` or `--validator valibot` is set. Lives
next to the model; never imported by the model itself.

### Zod variant (`--validator zod`)

```ts
/**
 * @generated by openapi-nexus. Do not edit.
 */
import { z } from 'zod';
import { CategorySchema } from './Category.schema';
import { PetStatusSchema } from './PetStatus.schema';
import { TagSchema } from './Tag.schema';

export const PetSchema = z.object({
  category: CategorySchema.nullable().optional(),
  id: z.number().int().nullable().optional(),
  name: z.string(),
  photoUrls: z.array(z.string()),
  status: PetStatusSchema.nullable().optional(),
  tags: z.array(TagSchema).nullable().optional(),
});

export const parsePet = (input: unknown) => PetSchema.parse(input);
export const safeParsePet = (input: unknown) => PetSchema.safeParse(input);
```

### Valibot variant (`--validator valibot`)

```ts
/**
 * @generated by openapi-nexus. Do not edit.
 */
import * as v from 'valibot';
import { CategorySchema } from './Category.schema';
import { PetStatusSchema } from './PetStatus.schema';
import { TagSchema } from './Tag.schema';

export const PetSchema = v.object({
  category: v.optional(v.nullable(CategorySchema)),
  id: v.optional(v.nullable(v.pipe(v.number(), v.integer()))),
  name: v.string(),
  photoUrls: v.array(v.string()),
  status: v.optional(v.nullable(PetStatusSchema)),
  tags: v.optional(v.nullable(v.array(TagSchema))),
});

export const parsePet = (input: unknown) => v.parse(PetSchema, input);
export const safeParsePet = (input: unknown) => v.safeParse(PetSchema, input);
```

**Why opt-in:** zod is ~14KB gz, valibot is ~2KB gz tree-shakable. Neither
belongs in a generated client by default. Users who need runtime validation opt
in; users who trust their server don't pay.

`package.json` gets `"peerDependencies": { "zod": "^3" }` (or
`"valibot": "^0.42"`) when the flag is set.

## 3. API file вЂ” `PetApi.ts`

### Before (709 lines for 8 operations)

- `ApiAddPetRequest`, `ApiDeletePetRequest`, ... вЂ” 8 wrapper interfaces (50 lines)
- `PetApiInterface` вЂ” duplicates every method signature (140 lines)
- `PetApi extends BaseAPI implements PetApiInterface` вЂ” both `xxxRaw` and `xxx` per operation (500 lines)
- `ApiXxxRequest` shapes and `xxxRaw` methods are noise the caller never wants.

### After (~220 lines target вЂ” 8 operations, single shape each)

```ts
/**
 * @generated by openapi-nexus. Do not edit.
 */
import type { Pet } from '../models/Pet';
import type { UploadResponse } from '../models/UploadResponse';
import { request, type ClientConfig } from '../runtime';

/** API client for pet operations */
export class PetApi {
  constructor(private readonly config: ClientConfig) {}

  /** Update an existing pet. */
  async updatePet(options: { body: Pet; signal?: AbortSignal }): Promise<Pet> {
    return request<Pet>(this.config, {
      method: 'PUT',
      path: '/pet',
      body: options.body,
      signal: options.signal,
    });
  }

  /** Add a new pet to the store. */
  async addPet(options: { body: Pet; signal?: AbortSignal }): Promise<Pet> {
    return request<Pet>(this.config, {
      method: 'POST',
      path: '/pet',
      body: options.body,
      signal: options.signal,
    });
  }

  /** Find pets by status. */
  async findPetsByStatus(options: { status: string; signal?: AbortSignal }): Promise<Pet[]> {
    return request<Pet[]>(this.config, {
      method: 'GET',
      path: '/pet/findByStatus',
      query: { status: options.status },
      signal: options.signal,
    });
  }

  /** Find pet by ID. */
  async getPetById(options: { petId: number; signal?: AbortSignal }): Promise<Pet> {
    return request<Pet>(this.config, {
      method: 'GET',
      path: `/pet/${encodeURIComponent(options.petId)}`,
      signal: options.signal,
    });
  }

  /** Delete a pet. */
  async deletePet(options: { petId: number; signal?: AbortSignal }): Promise<void> {
    await request<void>(this.config, {
      method: 'DELETE',
      path: `/pet/${encodeURIComponent(options.petId)}`,
      signal: options.signal,
    });
  }

  /** Upload an image. */
  async uploadFile(options: {
    petId: number;
    additionalMetadata?: string;
    body: Blob;
    signal?: AbortSignal;
  }): Promise<UploadResponse> {
    const form = new FormData();
    if (options.additionalMetadata !== undefined) form.append('additionalMetadata', options.additionalMetadata);
    form.append('file', options.body);
    return request<UploadResponse>(this.config, {
      method: 'POST',
      path: `/pet/${encodeURIComponent(options.petId)}/uploadImage`,
      body: form,
      signal: options.signal,
    });
  }

  // ... updatePetWithForm, findPetsByTags
}
```

**What's gone:**
- `ApiXxxRequest` wrapper interfaces вЂ” the options object is inlined at each method
- `PetApiInterface` вЂ” the class is the contract
- `xxxRaw` variants вЂ” callers who need the `Response` object can pass
  `{ raw: true }` (Phase 3 open question: keep this or drop; leaning toward
  drop and exposing `request()` as an escape hatch)
- Typed response unions (`JSONApiResponse<Pet> & { status: 200 } | ...`) вЂ” non-2xx
  responses throw `ApiResponseError` with `.status` and `.body` on the error

**What stays:**
- Per-tag class split (`PetApi`, `StoreApi`, `UserApi`)
- JSDoc from spec `summary` / `description` (one per method, not per `xxxRaw`/`xxx` pair)

## 4. Client wrapper вЂ” `Client.ts` (new)

Name derived from `info.title`. Petstore spec has `"Petstore API"` в†’ `PetstoreApiClient`.
The `"Studio AMS вЂ” ASM"` case: unicode-strip + `to_pascal_case` в†’ `StudioAmsAsm` +
`Client` suffix в†’ `StudioAmsAsmClient`.

```ts
/**
 * @generated by openapi-nexus. Do not edit.
 */
import type { ClientConfig } from './runtime';
import { PetApi } from './apis/PetApi';
import { StoreApi } from './apis/StoreApi';
import { UserApi } from './apis/UserApi';

export interface PetstoreApiClientOptions {
  baseUrl?: string;
  fetch?: typeof fetch;
  headers?: Record<string, string>;
  /** Called before every request. Return the (possibly modified) init. */
  beforeRequest?: (init: RequestInit & { url: string }) => RequestInit & { url: string } | Promise<RequestInit & { url: string }>;
}

export class PetstoreApiClient {
  readonly pets: PetApi;
  readonly store: StoreApi;
  readonly users: UserApi;

  constructor(options: PetstoreApiClientOptions = {}) {
    const config: ClientConfig = {
      baseUrl: options.baseUrl ?? 'https://petstore.example.com/api/v3',
      fetch: options.fetch ?? globalThis.fetch,
      headers: options.headers ?? {},
      beforeRequest: options.beforeRequest,
    };
    this.pets = new PetApi(config);
    this.store = new StoreApi(config);
    this.users = new UserApi(config);
  }
}
```

**Property naming:** tag в†’ lowercase + pluralize-if-needed. `pet` в†’ `pets`,
`store` в†’ `store`, `user` в†’ `users`. Phase 3 open question: should we always
singularize the class (`PetApi`) + pluralize the property (`pets`), or match
the tag exactly? Leaning toward tag-matches-property with `Api` suffix on class.

**CLI override:** `--client-name MyClient` produces `MyClient` regardless of
`info.title`.

## 5. Runtime вЂ” `runtime.ts`

### Before (450 lines)

`BaseAPI`, `Configuration`, `DefaultConfig`, `InitOverrideFunction`,
`JSONApiResponse`, `BlobApiResponse`, `TextApiResponse`, `VoidApiResponse`,
`ResponseError`, `RequiredError`, `exists()`, `querystring()`,
`mapValues()`, and a handful more. Most are unused at the call site.

### After (~80 lines target)

```ts
/**
 * @generated by openapi-nexus. Do not edit.
 */

export interface ClientConfig {
  baseUrl: string;
  fetch: typeof fetch;
  headers: Record<string, string>;
  beforeRequest?: (init: RequestInit & { url: string }) => RequestInit & { url: string } | Promise<RequestInit & { url: string }>;
}

export interface RequestOptions {
  method: string;
  path: string;
  query?: Record<string, string | number | boolean | undefined | null | ReadonlyArray<string | number | boolean>>;
  body?: unknown;
  headers?: Record<string, string>;
  signal?: AbortSignal;
}

export class ApiError extends Error {
  constructor(message: string, options?: { cause?: unknown }) {
    super(message, options);
    this.name = 'ApiError';
  }
}

export class ApiNetworkError extends ApiError {
  constructor(cause: unknown) {
    super('Network request failed', { cause });
    this.name = 'ApiNetworkError';
  }
}

export class ApiResponseError<T = unknown> extends ApiError {
  constructor(
    readonly status: number,
    readonly body: T,
    readonly response: Response,
  ) {
    super(`Request failed with status ${status}`);
    this.name = 'ApiResponseError';
  }
}

function buildQuery(query: RequestOptions['query']): string {
  if (!query) return '';
  const params = new URLSearchParams();
  for (const [k, v] of Object.entries(query)) {
    if (v === undefined || v === null) continue;
    if (Array.isArray(v)) {
      for (const item of v) params.append(k, String(item));
    } else {
      params.append(k, String(v));
    }
  }
  const s = params.toString();
  return s ? `?${s}` : '';
}

export async function request<T>(config: ClientConfig, options: RequestOptions): Promise<T> {
  const url = `${config.baseUrl}${options.path}${buildQuery(options.query)}`;
  const isForm = options.body instanceof FormData;
  const isBlob = options.body instanceof Blob;
  const hasBody = options.body !== undefined && options.body !== null;

  let init: RequestInit & { url: string } = {
    url,
    method: options.method,
    headers: {
      ...config.headers,
      ...(hasBody && !isForm && !isBlob ? { 'Content-Type': 'application/json' } : {}),
      ...options.headers,
    },
    body: hasBody
      ? (isForm || isBlob ? (options.body as BodyInit) : JSON.stringify(options.body))
      : undefined,
    signal: options.signal,
  };
  if (config.beforeRequest) init = await config.beforeRequest(init);

  let response: Response;
  try {
    response = await config.fetch(init.url, init);
  } catch (e) {
    throw new ApiNetworkError(e);
  }

  if (!response.ok) {
    const body = await response.clone().json().catch(() => response.text().catch(() => undefined));
    throw new ApiResponseError(response.status, body, response);
  }

  if (response.status === 204 || response.headers.get('content-length') === '0') return undefined as T;
  const ct = response.headers.get('content-type') ?? '';
  if (ct.includes('application/json')) return response.json() as Promise<T>;
  if (ct.startsWith('text/')) return response.text() as unknown as Promise<T>;
  return response.blob() as unknown as Promise<T>;
}
```

That's the whole runtime. No `BaseAPI`, no response wrappers, no `RequiredError`
(TS's type system catches missing required fields at compile time).

## 7. Go side (abbreviated)

Go keeps its current overall shape вЂ” per-tag client structs, positional params,
struct + method receiver idiom. Scope here is:

- Drop the `openapi-nexus-go-http/templates/` Jinja files; move to sigil-stitch
- Keep `encoding/json` based marshaling (no validator story вЂ” Go users have `validator/v10`, `ozzo-validation`, etc.; not our job)
- Slim the runtime similarly (drop any unused abstractions)
- Same `info.title`-derived client naming

Concrete Go target goes in a follow-up spec doc during Phase 4 once the TS
migration proves the sigil-stitch approach. No breaking changes to Go output
planned beyond what falls out of the template в†’ sigil-stitch rewrite.

## 8. Migration checklist for users

When v0.2.0 ships, the release notes point at this table:

| Old | New |
|---|---|
| `import { Pet, PetFromJSON } from 'petstore/models/Pet'` | `import type { Pet } from 'petstore/models/Pet'` |
| `PetFromJSON(json)` | direct assign (data is structural) or `parsePet(json)` with `--validator` |
| `instanceOfPet(x)` | `parsePet(x)` (with validator) or write your own guard |
| `new PetApi(config).getPetByIdRaw({ petId: 7 })` | `new PetstoreApiClient({ baseUrl }).pets.getPetById({ petId: 7 })` |
| `try { ... } catch (e: ResponseError) { e.response }` | `try { ... } catch (e) { if (e instanceof ApiResponseError) e.body }` |
| `import { Pet } from 'petstore'` (via barrel) | `import type { Pet } from 'petstore/models/Pet'` (or re-export from root if we keep that) |

## 9. Open questions (punt to Phase 3)

1. **Raw response escape hatch** вЂ” drop `xxxRaw` entirely, or keep as `{ raw: true }` flag on the options object? Leaning drop + expose `request()` directly.
2. **Property pluralization** вЂ” `pets` vs `pet` on the `Client`. Match tag exactly, or prettify?
3. **Form data detection** вЂ” do we sniff `application/x-www-form-urlencoded` vs `multipart/form-data` from the spec's `requestBody.content`? Current path assumes `multipart` for anything with a `Blob`. Needs spec survey.
4. **Streaming responses** вЂ” `Response.body` passthrough for `application/octet-stream`? Probably yes вЂ” change `request<T>` return to allow `Response` when caller asks.
5. **Deprecated operations** вЂ” emit `@deprecated` JSDoc + TS `/** @deprecated */` tag? Cheap win; add to Phase 3.

These don't block Phase 1 (Go on IR pipeline) or Phase 2 (sigil-stitch spike).
Revisit before finalizing Phase 3 golden updates.
